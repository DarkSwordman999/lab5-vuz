 База данных ВУЗ

## О проекте
Реляционная база данных для учёта успеваемости студентов высшего учебного заведения.
Проект выполнен в рамках курса по базам данных. Содержит полный цикл работы с БД:
от создания таблиц до сложных аналитических запросов.

## Технологии
- **PostgreSQL 16** - основная СУБД
- **Bash** - скрипты для автоматизации
- **Git** - контроль версий
- **pgdump/psql** - работа с дампом и выполнение запросов

## Структура базы данных
### Таблицы
| Таблица       | Описание                                                  | Количество записей |
|---------------|-----------------------------------------------------------|--------------------|
| СТУДЕНТЫ      | Информация о студентах (ФИО, телефон, email)              |        52          |
| ГРУППЫ        | Учебные группы (название, курс, факультет)                |        10          |
| ПРЕПОДАВАТЕЛИ | Данные преподавателей (ФИО, степень, должность)           |        15          |
| ДИСЦИПЛИНЫ    | Учебные дисциплины (название, часы, семестр)              |        21          |
| УСПЕВАЕМОСТЬ  | Оценки студентов (связь студент-дисциплина-преподаватель) |       1000+        |
| СЕЗОН         | Сезоны для аналитики (месяц → сезон)                      |        12          |

### Схема связей
- СТУДЕНТЫ → ГРУППЫ (многие к одному)
- УСПЕВАЕМОСТЬ → СТУДЕНТЫ (многие к одному)
- УСПЕВАЕМОСТЬ → ДИСЦИПЛИНЫ (многие к одному)
- УСПЕВАЕМОСТЬ → ПРЕПОДАВАТЕЛИ (многие к одному)
- УСПЕВАЕМОСТЬ → ГРУППЫ (многие к одному)

## Прикладные задачи (по заданию)

### Задача 1: Распределение среднего балла по временам года
```sql
SELECT 
    СЕЗОН.сезон AS "Время года",
    AVG(УСПЕВАЕМОСТЬ.оценка) AS "Средний балл"
FROM УСПЕВАЕМОСТЬ
INNER JOIN СЕЗОН ON СЕЗОН.месяц = EXTRACT(MONTH FROM УСПЕВАЕМОСТЬ.дата)
GROUP BY СЕЗОН.сезон, СЕЗОН.порядок
ORDER BY СЕЗОН.порядок;
```

### Задача 2: Распределение среднего балла по факультетам и сезонам
```sql
SELECT 
    СЕЗОН.сезон AS "Время года",
    ГРУППЫ.факультет AS "Факультет",
    AVG(УСПЕВАЕМОСТЬ.оценка) AS "Средний балл"
FROM УСПЕВАЕМОСТЬ
INNER JOIN ГРУППЫ ON ГРУППЫ.код = УСПЕВАЕМОСТЬ.группа
INNER JOIN СЕЗОН ON СЕЗОН.месяц = EXTRACT(MONTH FROM УСПЕВАЕМОСТЬ.дата)
GROUP BY СЕЗОН.сезон, СЕЗОН.порядок, ГРУППЫ.факультет
ORDER BY СЕЗОН.порядок, ГРУППЫ.факультет;
```

### Задача 3: Распределение среднего балла по дням недели и преподавателям
```sql
SELECT 
    EXTRACT(DOW FROM УСПЕВАЕМОСТЬ.дата) AS "День недели",
    ПРЕПОДАВАТЕЛИ.фамилия || ' ' || SUBSTRING(ПРЕПОДАВАТЕЛИ.имя, 1, 1) || '.' AS "Преподаватель",
    AVG(УСПЕВАЕМОСТЬ.оценка) AS "Средний балл"
FROM УСПЕВАЕМОСТЬ
INNER JOIN ПРЕПОДАВАТЕЛИ ON ПРЕПОДАВАТЕЛИ.код = УСПЕВАЕМОСТЬ.преподаватель
GROUP BY "День недели", ПРЕПОДАВАТЕЛИ.фамилия, ПРЕПОДАВАТЕЛИ.имя
ORDER BY "День недели", "Средний балл" DESC;
```

### Задача 4: Распределение среднего балла по дисциплинам для заданного дня недели
```sql
-- Использование: psql -d vuz_lab5 -v arg1=1
SELECT 
    ДИСЦИПЛИНЫ.название AS "Дисциплина",

```
ТИПЫ JOIN В ПРОЕКТЕ
================================================================================

1. INNER JOIN (./h 07)
Назначение: Возвращает только те записи, у которых есть совпадение в обеих
таблицы. Показывает только данные, где связь существует.

SQL-запрос:
--------------------------------------------------------------------------------
```sql
SELECT 
    У.код_записи,
    С.фамилия || ' ' || С.имя AS студент,
    Д.название AS дисциплина,
    У.оценка
FROM УСПЕВАЕМОСТЬ У
INNER JOIN СТУДЕНТЫ С ON С.код = У.студент
INNER JOIN ДИСЦИПЛИНЫ Д ON Д.код = У.дисциплина
LIMIT 10;
```
Пример результата:
Когда использовать:
- Нужны только оценки, у которых есть и студент, и дисциплина
- Анализ успеваемости по существующим студентам
- Формирование отчетов без "пустых" записей

2. LEFT JOIN (./h 08)
Назначение: Возвращает ВСЕ записи из левой таблицы (ДИСЦИПЛИНЫ) и совпадающие
записи из правой таблицы (УСПЕВАЕМОСТЬ). Если совпадений нет - NULL.

SQL-запрос:
--------------------------------------------------------------------------------
```sql
SELECT 
    Д.название,
    COUNT(У.код_записи) AS количество_оценок,
    AVG(У.оценка) AS средний_балл
FROM ДИСЦИПЛИНЫ Д
LEFT JOIN УСПЕВАЕМОСТЬ У ON Д.код = У.дисциплина
GROUP BY Д.название
ORDER BY количество_оценок DESC
LIMIT 10;
```
Пример результата:
Когда использовать:
- Показать все дисциплины, даже если по ним нет оценок
- Анализ дисциплин, которые никто не сдавал
- Поиск "проблемных" дисциплин без успеваемости

3. CROSS JOIN (./h 09)
Назначение: Декартово произведение. Каждая строка из первой таблицы соединяется
с каждой строкой из второй таблицы.

SQL-запрос:
--------------------------------------------------------------------------------
```sql
SELECT 
    С.фамилия || ' ' || С.имя AS студент,
    Д.название AS дисциплина,
    У.оценка,
    У.дата
FROM УСПЕВАЕМОСТЬ У
JOIN СТУДЕНТЫ С ON С.код = У.студент
JOIN ДИСЦИПЛИНЫ Д ON Д.код = У.дисциплина
ORDER BY У.дата DESC
LIMIT 10;
```
Пример результата:
Когда использовать:
- Получение полной информации о каждой оценке
- Быстрое соединение всех связанных таблиц
- Формирование детальных отчетов


4. RIGHT JOIN (в проекте не используется, но синтаксис)
```sql
SELECT * FROM ТаблицаA RIGHT JOIN ТаблицаB ON условие
```
- Аналогичен LEFT JOIN, но таблицы меняются местами

5. FULL JOIN (в проекте не используется, но синтаксис)
```sql
SELECT * FROM ТаблицаA FULL JOIN ТаблицаB ON условие
```
- Возвращает все записи из обеих таблиц

6. SELF JOIN (в проекте не используется, но синтаксис)
```sql
SELECT * FROM Таблица A JOIN Таблица B ON A.поле = B.поле
```
- Таблица соединяется сама с собой

## Группа 1: Подзапросы в роли ПОЛЕЙ
## Задача 1.1 - Отклонение среднего балла студента от среднего по группе
```sql
SELECT 
    СТ.фамилия || ' ' || СТ.имя AS студент,
    ГР.название AS группа,
    (SELECT AVG(оценка) FROM УСПЕВАЕМОСТЬ WHERE студент = СТ.код) AS средний_балл_студента,
    (SELECT AVG(У.оценка) FROM УСПЕВАЕМОСТЬ У WHERE У.группа = ГР.код) AS средний_балл_группы,
    ROUND(...) AS отклонение
FROM СТУДЕНТЫ СТ
CROSS JOIN ГРУППЫ ГР
ORDER BY отклонение DESC
LIMIT 10;
```
## Задача 1.2 - Рейтинг преподавателей
```sql
SELECT 
    ПР.фамилия || ' ' || ПР.имя AS преподаватель,
    (SELECT AVG(оценка) FROM УСПЕВАЕМОСТЬ WHERE преподаватель = ПР.код) AS средний_балл,
    (SELECT AVG(оценка) FROM УСПЕВАЕМОСТЬ) AS общий_средний,
    ROUND(...) AS отклонение
FROM ПРЕПОДАВАТЕЛИ ПР
WHERE (SELECT COUNT(*) FROM УСПЕВАЕМОСТЬ WHERE преподаватель = ПР.код) > 0
ORDER BY средний_балл DESC
LIMIT 10;
```

## Задача 2.1 - Студенты со средним баллом выше среднего по дисциплине
```sql
SELECT *
FROM (
    SELECT 
        СТ.фамилия || ' ' || СТ.имя AS студент,
        Д.название AS дисциплина,
        AVG(У.оценка) AS средний_балл_студента,
        (SELECT AVG(У2.оценка) FROM УСПЕВАЕМОСТЬ У2 WHERE У2.дисциплина = Д.код) AS средний_балл_дисциплины
    FROM УСПЕВАЕМОСТЬ У
    JOIN СТУДЕНТЫ СТ ON СТ.код = У.студент
    JOIN ДИСЦИПЛИНЫ Д ON Д.код = У.дисциплина
    GROUP BY СТ.фамилия, СТ.имя, Д.название, Д.код
) AS subquery
WHERE средний_балл_студента > средний_балл_дисциплины
ORDER BY дисциплина, средний_балл_студента DESC
LIMIT 20;
```

## Группа 3: Подзапросы в роли СВЯЗАННЫХ ТАБЛИЦ (LATERAL)
## Задача 3.1 - Топ-3 лучших студента на каждом факультете
```sql
SELECT 
    ф.факультет,
    top.студент,
    top.средний_балл,
    top.место
FROM (SELECT DISTINCT факультет FROM ГРУППЫ) ф,
LATERAL (
    SELECT 
        СТ.фамилия || ' ' || СТ.имя AS студент,
        ROUND(AVG(У.оценка), 2) AS средний_балл,
        ROW_NUMBER() OVER (ORDER BY AVG(У.оценка) DESC) AS место
    FROM СТУДЕНТЫ СТ
    JOIN УСПЕВАЕМОСТЬ У ON У.студент = СТ.код
    JOIN ГРУППЫ ГР ON ГР.код = У.группа
    WHERE ГР.факультет = ф.факультет
    GROUP BY СТ.фамилия, СТ.имя
    ORDER BY средний_балл DESC
    LIMIT 3
) AS top
ORDER BY ф.факультет, top.место;
```

## Задача 4.1 - Оценки выше среднего по дисциплине (с параметром)
```sql 
-- Использование: ./h 115 "Базы данных"
SELECT 
    СТ.фамилия || ' ' || СТ.имя AS студент,
    Д.название AS дисциплина,
    У.оценка,
    (SELECT AVG(У2.оценка) FROM УСПЕВАЕМОСТЬ У2 WHERE У2.дисциплина = Д.код) AS средний_балл_по_дисциплине
FROM УСПЕВАЕМОСТЬ У
JOIN СТУДЕНТЫ СТ ON СТ.код = У.студент
JOIN ДИСЦИПЛИНЫ Д ON Д.код = У.дисциплина
WHERE Д.название = :arg1
  AND У.оценка > (SELECT AVG(У2.оценка) FROM УСПЕВАЕМОСТЬ У2 WHERE У2.дисциплина = Д.код)
ORDER BY У.оценка DESC;
```
## Задача 4.2 - Студенты, сдавшие все экзамены на 4 и 5 (NOT EXISTS)
```sql
SELECT 
    СТ.фамилия || ' ' || СТ.имя AS студент,
    COUNT(У.оценка) AS количество_экзаменов,
    ROUND(AVG(У.оценка), 2) AS средний_балл
FROM СТУДЕНТЫ СТ
JOIN УСПЕВАЕМОСТЬ У ON У.студент = СТ.код
WHERE NOT EXISTS (
    SELECT 1 FROM УСПЕВАЕМОСТЬ У2 
    WHERE У2.студент = СТ.код AND У2.оценка < 4
)
GROUP BY СТ.фамилия, СТ.имя, СТ.код
ORDER BY средний_балл DESC
LIMIT 20;
```
## Задача 4.3 - Студенты с минимальной оценкой выше общего среднего
```sql
SELECT 
    СТ.фамилия || ' ' || СТ.имя AS студент,
    ROUND(AVG(У.оценка), 2) AS средний_балл,
    COUNT(У.оценка) AS количество_оценок,
    MIN(У.оценка) AS мин_оценка,
    MAX(У.оценка) AS макс_оценка
FROM СТУДЕНТЫ СТ
JOIN УСПЕВАЕМОСТЬ У ON У.студент = СТ.код
GROUP BY СТ.фамилия, СТ.имя, СТ.код
HAVING MIN(У.оценка) > (SELECT AVG(оценка) FROM УСПЕВАЕМОСТЬ)
ORDER BY средний_балл DESC
LIMIT 20;
```

## Функции пользователя (PL/pgSQL)

### Задача 1: Распределение студентов по среднему баллу (один признак)

**Постановка:** Требуется определить студентов, чей средний балл по всем дисциплинам выше заданного порогового значения.

**Используемые таблицы и поля:** СТУДЕНТЫ (код, фамилия, имя), УСПЕВАЕМОСТЬ (студент, оценка)

**Результат:** таблица с колонками: студент_код, фамилия, имя, средний_балл, количество_оценок

**Функции:**

#### Функция 1: `get_student_avg_grade(INT)`
- **Тип:** скалярная (возвращает одно значение)
- **Аргумент:** код студента
- **Возвращает:** средний балл студента (NUMERIC)

```sql
CREATE OR REPLACE FUNCTION get_student_avg_grade(p_student_code INT)
RETURNS NUMERIC(3,2) AS $$
BEGIN
    RETURN (
        SELECT ROUND(AVG(оценка), 2)
        FROM УСПЕВАЕМОСТЬ
        WHERE студент = p_student_code
    );
END;
$$ LANGUAGE PLpgSQL;
```
#### Функция 2: `students_above_grade(NUMERIC)`
- **Тип:** табличная (SETOF)
- **Аргумент:** минимальный средний балл (по умолчанию 4.0)
- **Возвращает:** таблицу студентов


### Задача 3: Сводная таблица (группы × дисциплины)
#### Функция 1 `get_avg_grade_for_pair(VARCHAR, VARCHAR)`

```sql
CREATE OR REPLACE FUNCTION get_avg_grade_for_pair(
    p_group_name VARCHAR,
    p_subject_name VARCHAR
) RETURNS NUMERIC(3,2) AS $$
BEGIN
    RETURN (
        SELECT ROUND(AVG(У.оценка), 2)
        FROM УСПЕВАЕМОСТЬ У
        JOIN ГРУППЫ Г ON Г.код = У.группа
        JOIN ДИСЦИПЛИНЫ Д ON Д.код = У.дисциплина
        WHERE Г.название = p_group_name 
          AND Д.название = p_subject_name
    );
END;
$$ LANGUAGE PLpgSQL;
```

#### Функция 4: `build_pivot_table()`

```sql
CREATE OR REPLACE FUNCTION build_pivot_table()
RETURNS TABLE(group_name VARCHAR, pivot_data JSONB) AS $$
DECLARE
    v_group_name VARCHAR;
    v_subject_name VARCHAR;
    v_result JSONB := '{}'::JSONB;
BEGIN
    FOR v_group_name IN SELECT name FROM get_all_groups() LOOP
        v_result := '{}'::JSONB;
        FOR v_subject_name IN SELECT name FROM get_all_subjects() LOOP
            v_result := v_result || jsonb_build_object(
                v_subject_name,
                COALESCE(get_avg_grade_for_pair(v_group_name, v_subject_name), 0)
            );
        END LOOP;
        RETURN QUERY SELECT v_group_name, v_result;
    END LOOP;
END;
$$ LANGUAGE PLpgSQL;
```

